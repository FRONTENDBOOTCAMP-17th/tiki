# 이벤트 상세 메타데이터 · 동적 OG 이미지 (`bang/feat/final-sprint`)

> 배경: 그동안 모든 페이지가 `layout.tsx`의 기본 타이틀을 공유했습니다. 이벤트 상세를 카톡/SNS에 공유하면 제목·설명·미리보기 카드가 이벤트별로 표시되도록 추가했습니다.

## 1. 변경/추가 파일

| 경로 | 내용 |
|---|---|
| `src/app/[eventId]/page.tsx` | `generateMetadata` 추가, `getEventDetail`을 `cache()`로 감쌈 |
| `src/app/[eventId]/opengraph-image.tsx` (신규) | `next/og`로 1200×630 미리보기 카드 생성 |

## 2. generateMetadata

- `getEventDetail`을 재사용하되, page 본문과 중복 조회되지 않도록 React `cache()`로 감쌌습니다. 같은 요청 안에서는 한 번만 조회됩니다.
- 이벤트가 있으면 제목·설명(앞 100자)·openGraph 정보를, 형식이 맞지 않거나 없으면 기본 메타데이터를 반환합니다. 여기서 예외를 던지지 않고, `notFound()`는 page가 담당합니다.
- 조회가 실패해도 기본 메타데이터로 폴백해, 메타데이터 때문에 페이지가 중단되지 않게 했습니다.

## 3. 동적 OG 이미지

`opengraph-image.tsx`는 파일 컨벤션이라, 이 파일이 있으면 Next가 `og:image`를 자동으로 연결합니다. 미리보기 이미지는 이 파일 하나를 단일 소스로 삼아 관리합니다.

- 포스터 + 제목 + 공연 기간/장소를 브랜드 컬러(`#9B6FC0` / `#332A40` / `#F6F1FA`) 카드로 그립니다.
- 데이터 조회에 실패하면 로고와 서비스명만 있는 기본 카드로 폴백합니다.

## 4. 렌더 확인

포스터가 `sharp`로 변환된 webp라, 실제 OG 이미지 라우트를 직접 호출해 확인했습니다. 실존 이벤트(webp 포스터)와 폴백 카드(로고) 모두 `200 image/png`로 정상 렌더됩니다(Next 16 기준).

## 5. 앞으로

- OG 카드에는 제목·기간·장소까지 담았습니다. 필요하면 요약 한 줄을 더 넣을 수 있습니다.
- 폴백 카드의 로고 URL은 `NEXT_PUBLIC_SITE_URL`이 없으면 배포 도메인을 기본값으로 사용합니다. 도메인이 바뀌면 이 값을 확인하면 됩니다.
