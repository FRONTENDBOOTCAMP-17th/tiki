# TiKi 16차 해결 가이드 (2026-07-01)

이번 [필수]는 SEO와 결제 취소 경로 마무리, 이렇게 둘입니다. SEO는 파일 하나 고치고 두 개 만들면 끝나는 일이라 여기부터 잡겠습니다.

---

## 1. [필수] SEO — layout 메타 + sitemap + robots

### (1) `src/app/layout.tsx` 메타데이터 교체

지금:
```tsx
export const metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
// ...
<html lang="en">
```

이렇게 바꿉니다:
```tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  metadataBase: new URL("https://tiki.example.com"), // 실제 배포 도메인으로
  title: {
    default: "TiKi — 티켓 예약 오픈마켓",
    template: "%s | TiKi",
  },
  description: "콘서트·뮤지컬·전시 등 다양한 이벤트 티켓을 예약하는 오픈마켓",
  openGraph: {
    title: "TiKi — 티켓 예약 오픈마켓",
    description: "다양한 이벤트 티켓을 한곳에서",
    type: "website",
  },
};
// ...
<html lang="ko">
```

`title.template`을 넣어 두면 이벤트 상세 페이지에서 `generateMetadata`로 공연 제목만 돌려줘도 알아서 `공연이름 | TiKi` 꼴이 됩니다.

### (2) 이벤트 상세에 `generateMetadata` 추가

`src/app/[eventId]/page.tsx`(서버 컴포넌트)에 함수 하나만 export 하면 됩니다.

```tsx
export async function generateMetadata(
  { params }: { params: Promise<{ eventId: string }> }
): Promise<Metadata> {
  const { eventId } = await params;
  // UUID 형식 아니면 조회하지 말고 기본값 (아래 3번 가드와 같은 정규식)
  if (!/^[0-9a-f-]{36}$/i.test(eventId)) return { title: "공연 정보" };
  const event = await getEventDetail(eventId);
  if (!event) return { title: "존재하지 않는 공연" };
  return {
    title: event.title,
    description: event.summary ?? undefined,
    openGraph: { title: event.title, images: event.poster_url ? [event.poster_url] : [] },
  };
}
```

### (3) `src/app/sitemap.ts` 생성

```ts
import type { MetadataRoute } from "next";
import { createClient } from "@/lib/supabase/server"; // 프로젝트 서버 클라이언트

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const base = "https://tiki.example.com";
  const supabase = await createClient();
  const { data: events } = await supabase
    .from("event")
    .select("id, updated_at")
    .eq("status", "공개"); // 공개 상태만

  const eventUrls = (events ?? []).map((e) => ({
    url: `${base}/${e.id}`,
    lastModified: e.updated_at ?? undefined,
  }));

  return [
    { url: base, changeFrequency: "daily", priority: 1 },
    { url: `${base}/category`, changeFrequency: "daily", priority: 0.8 },
    ...eventUrls,
  ];
}
```

### (4) `src/app/robots.ts` 생성

```ts
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: "*", allow: "/", disallow: ["/admin", "/mypage"] },
    sitemap: "https://tiki.example.com/sitemap.xml",
  };
}
```

> 확인: dev에서 `curl localhost:3120/sitemap.xml`, `curl localhost:3120/robots.txt`를 쳐서 200에 XML/텍스트가 나오면 된 겁니다. `<html lang="ko">`도 view-source로 같이 확인해 주세요.

## 2. [필수] 결제 취소 경로 마무리 — PaymentForm.tsx:107

데스크톱 경로(`:158`)엔 이미 `await`를 붙여 두셨고, 이제 리다이렉트 복귀 경로(`:107`) 하나만 남았습니다.

```tsx
// PaymentForm.tsx  (리다이렉트 복귀 후 취소 처리, effect 콜백 안)
useEffect(() => {
  if (canceled) {
    // effect 콜백은 async가 안 되니, 실패를 삼키지 않게 void + catch
    void cancelReservation(orderId).catch((e) => {
      console.error("예약 취소 실패:", e);
    });
  }
}, [canceled, orderId]);
```

핵심은 `cancelReservation(orderId)`를 그냥 호출만 하고 끝내지 말고 실패했을 때 로그라도 남도록 `.catch`를 붙이는 건데, 지금 구조에서는 취소가 실패해도 아무도 모르게 조용히 지나가 버리기 때문입니다.

## 3. [제안] 비-UUID eventId 500 막기

`src/app/[eventId]/page.tsx` 맨 위에서 형식부터 검사해 주면 됩니다.

```tsx
import { notFound } from "next/navigation";

const UUID_RE = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;

export default async function EventPage({ params }: { params: Promise<{ eventId: string }> }) {
  const { eventId } = await params;
  if (!UUID_RE.test(eventId)) notFound(); // DB에 uuid 아닌 값을 안 보냄

  const event = await getEventDetail(eventId);
  if (!event) notFound();
  // ...
}
```

이렇게 하면 봇이나 오타로 들어온 URL이 500 대신 깔끔하게 404로 빠집니다. seller·admin의 `[id]` 라우트에도 같은 정규식을 쓰면 되는데, 공통 유틸 `isUuid(s)`로 빼 두면 더 낫습니다.

## 4. [제안] 수량 주문 상한 통일

좌석 경로엔 8석 캡이 걸려 있는데 정작 수량 경로(`api/orders/route.ts`)엔 그게 빠져 있습니다.

```ts
const MAX_QTY_PER_ORDER = 8; // 좌석 경로와 같은 값
if (quantity < 1 || quantity > MAX_QTY_PER_ORDER) {
  return NextResponse.json({ error: "1회 최대 8매까지 구매할 수 있습니다." }, { status: 400 });
}
```

상수를 `lib/constants.ts` 한 곳에 두고 좌석·수량 양쪽이 그 값을 같이 참조하게 해 두면, 나중에 정책이 바뀌어도 한 군데만 고치면 됩니다.
