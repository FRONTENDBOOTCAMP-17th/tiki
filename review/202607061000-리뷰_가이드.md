## 3. `[제안]` CSV 수식 인젝션 방어 (방효진, 15분)

`orders/export/route.ts`의 `csvCell`은 콤마·따옴표는 잘 감쌉니다. 그런데 셀 값이 `=`, `+`, `-`, `@`로 시작하면 엑셀/구글시트가 **그걸 수식으로 실행**합니다. 예를 들어 어떤 구매자가 이름을 `=1+1`이나 `=HYPERLINK("...")`로 넣으면, 판매자가 정산 CSV를 열었을 때 그 수식이 돌아버립니다. 사용자 입력(이벤트명·구매자 이름 등)이 그대로 셀에 들어가니 막아두는 게 안전합니다.

해결은 "위험한 첫 글자면 앞에 작은따옴표를 붙여 문자열로 강제"하면 됩니다.

```ts
// 콤마/따옴표/줄바꿈 처리 + 수식 인젝션 방어
function csvCell(value: unknown) {
  let text = value == null ? "" : String(value);
  // = + - @ 로 시작하면 수식으로 해석되므로 작은따옴표로 무력화
  if (/^[=+\-@]/.test(text)) {
    text = "'" + text;
  }
  return /[",\r\n]/.test(text) ? `"${text.replace(/"/g, '""')}"` : text;
}
```

## 4. `[사소]` OAuth 콜백 이중슬래시 (`//auth/callback`)

`NEXT_PUBLIC_SITE_URL`이 끝에 슬래시가 붙어 있으면(`http://localhost:3000/`), `login/action.ts`의

```ts
const callbackUrl = new URL(`${process.env.NEXT_PUBLIC_SITE_URL}/auth/callback`);
```

가 `.../*/auth/callback`처럼 `//`를 만듭니다. 두 방법 중 하나로 정리하면 됩니다.

- 환경변수에서 끝 슬래시를 빼거나(가장 간단), 
- 코드에서 base와 경로를 `new URL`로 합칩니다.

```ts
// 끝 슬래시가 있어도 안전하게
const callbackUrl = new URL('/auth/callback', process.env.NEXT_PUBLIC_SITE_URL);
```

`new URL(path, base)` 형태는 base 끝 슬래시 유무와 상관없이 항상 깔끔한 경로를 만들어 줍니다. sitemap.ts에서 이미 이 방식을 쓰고 있으니 동일하게 맞추면 됩니다.

## 7. 발표 데모 동선 굳히기 (전원)

발표에서 스태프 기능이 이 조의 차별점이라, 아래 한 줄기를 미리 고정해 두면 흔들림 없이 보여줄 수 있습니다.

1. (판매자) 로그인 → `/seller/staff`에서 스태프 이메일 초대
2. (스태프) 로그인 → `/staff`에서 초대 수락 → 담당 공연 진입
3. (구매자) 이벤트 예매 → 결제 → 마이페이지에서 QR 확인
4. (스태프) 체크인 화면에서 그 QR 스캔(또는 코드 직접 입력) → 입장 확정
5. 같은 QR 다시 스캔 → "이미 사용됨" 뜨는 것까지

3~4번이 실제로 이어지는지(주문의 QR을 스태프가 스캔했을 때 `checkin_ticket`이 `ok`를 주는지)는 오늘내일 중 실계정으로 한 번 끝까지 돌려보길 권합니다. 이선우 님의 좌석·결제 흐름과 김연수 님의 체크인이 만나는 지점이라, 여기만 매끄러우면 데모가 삽니다.
