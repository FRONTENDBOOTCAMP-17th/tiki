## 1. proxy 미들웨어 — 세션 쿠키를 브라우저로 돌려보내기 [필수]

### 왜 중요한가

`@supabase/ssr`에서 세션은 쿠키에 들어 있습니다. 토큰은 보통 한 시간이면 만료되는데, 이때 누군가는 새 토큰을 받아 쿠키를 갱신해줘야 합니다. 그 역할을 하는 곳이 **미들웨어**입니다.

핵심은 "누가 쿠키를 쓸 수 있느냐"입니다.
- **서버 컴포넌트**는 쿠키를 쓸 수 없습니다(읽기만). 그래서 `supabase.server.ts`의 `setAll`은 일부러 try/catch로 감싸 조용히 넘어가게 돼 있습니다 — 미들웨어가 대신 갱신해준다는 전제니까요.
- **미들웨어**는 쿠키를 쓸 수 있습니다. 그래서 갱신은 여기서 일어나야 합니다.

지금 `proxy.ts`는 서버 컴포넌트용 `createClient()`를 그대로 가져다 썼습니다. 그러면 갱신된 쿠키를 쓰는 코드가 try/catch에 삼켜져, **새 토큰이 브라우저로 안 나갑니다.** 결국 토큰 만료 시점에 세션이 슬그머니 끊깁니다.

### 어떻게 고치나

미들웨어 전용 클라이언트를 따로 만듭니다. 핵심은 `setAll`에서 `request`와 `response` **양쪽에** 쿠키를 쓰고, **그 response를 반환**하는 것입니다.

```ts
// proxy.ts
import { createServerClient } from "@supabase/ssr";
import { NextResponse, type NextRequest } from "next/server";

export async function proxy(request: NextRequest) {
  let response = NextResponse.next({ request });

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!, // 또는 publishable key
    {
      cookies: {
        getAll() {
          return request.cookies.getAll();
        },
        setAll(cookiesToSet) {
          // 1) request에 반영(이후 코드가 최신 쿠키를 보게)
          cookiesToSet.forEach(({ name, value }) =>
            request.cookies.set(name, value)
          );
          // 2) response를 새로 만들고
          response = NextResponse.next({ request });
          // 3) response에도 써서 브라우저로 내보냄
          cookiesToSet.forEach(({ name, value, options }) =>
            response.cookies.set(name, value, options)
          );
        },
      },
    }
  );

  // 이 줄이 만료 토큰을 갱신하고 setAll을 호출하게 만듭니다.
  await supabase.auth.getUser();

  return response; // 갱신된 쿠키가 실린 response를 반드시 반환
}
```

마지막 `return response`가 빠지면 갱신해놓고도 안 내보내는 셈이니, 거기까지 꼭 챙기세요.

### (선택) 인증 가드도 같은 곳에서 [제안]

보호 라우트가 늘면 페이지마다 막는 것보다 미들웨어 한곳에서 막는 게 빠짐이 없습니다.

```ts
  const { data: { user } } = await supabase.auth.getUser();

  const path = request.nextUrl.pathname;
  const isPublic = path.startsWith("/login") || path.startsWith("/auth");
  if (!user && !isPublic) {
    const url = request.nextUrl.clone();
    url.pathname = "/login";
    return NextResponse.redirect(url);
  }
```

`matcher`로 정적 파일은 제외하는 것도 잊지 마세요(이미지·`_next` 등).

## 2. Toast 더블 브래킷 오타 [필수]

`Toast.styles.ts:11`에서 `]`가 하나 더 붙어 있습니다.

```ts
// before
"text-[#3A6BA5]]"
// after
"text-[#3A6BA5]"
```

`tsc`/`build`는 문자열이라 통과하지만, Tailwind는 `text-[#3A6BA5]]`를 유효한 클래스로 못 읽어 **그냥 무시**합니다. 그래서 info 토스트 글자색이 안 먹습니다. 고친 뒤 info 토스트를 한 번 띄워 색이 들어오는지 눈으로 확인해주세요.

## 3. CI에 push 트리거 + lint 추가 [제안]

### push 트리거 — 머지 후에도 green 보장

지금은 PR을 열 때만 CI가 돕니다. PR이 통과해도, 머지하는 순간 다른 PR과 합쳐지며 깨질 수 있는데 그건 안 잡힙니다. `push`도 같이 거세요.

```yaml
on:
  pull_request:
    branches: [develop]
  push:
    branches: [develop]   # 머지 후 develop도 검사
```

### lint 단계 — exhaustive-deps 등 잡기

`tsc`는 타입만 봅니다. `useEffect` 의존성 누락 같은 건 eslint가 잡습니다.

```yaml
      - run: npm ci
      - run: npm run lint        # 추가
      - run: npx tsc --noEmit
      - run: npm run build
```

`npm run lint`가 `package.json`에 없으면 `"lint": "next lint"`(또는 eslint 직접 호출)를 먼저 추가하세요.

## 4. Modal 키보드 접근성 — 트랩/복원/labelledby [제안]

Modal 한 곳만 고치면 이걸 재사용하는 모든 모달이 같이 좋아집니다.

```tsx
function Modal({ open, onClose, labelledBy, children }: ModalProps) {
  const ref = useRef<HTMLDivElement>(null);
  const prevFocus = useRef<HTMLElement | null>(null);

  useEffect(() => {
    if (!open) return;

    // 1) 닫을 때 돌아올 곳 기억
    prevFocus.current = document.activeElement as HTMLElement;

    // 2) 모달 안 첫 포커스 요소로 이동
    const focusables = ref.current?.querySelectorAll<HTMLElement>(
      'a[href], button:not([disabled]), textarea, input, select, [tabindex]:not([tabindex="-1"])'
    );
    focusables?.[0]?.focus();

    // 3) Tab을 모달 안에 가두기
    function onKey(e: KeyboardEvent) {
      if (e.key === "Escape") return onClose();
      if (e.key !== "Tab" || !focusables || focusables.length === 0) return;
      const first = focusables[0];
      const last = focusables[focusables.length - 1];
      if (e.shiftKey && document.activeElement === first) {
        e.preventDefault();
        last.focus();
      } else if (!e.shiftKey && document.activeElement === last) {
        e.preventDefault();
        first.focus();
      }
    }
    document.addEventListener("keydown", onKey);

    return () => {
      document.removeEventListener("keydown", onKey);
      // 4) 닫을 때 원래 자리로 포커스 복원
      prevFocus.current?.focus();
    };
  }, [open, onClose]);

  if (!open) return null;

  return createPortal(
    <div className="backdrop" onClick={onClose}>
      <div
        ref={ref}
        role="dialog"
        aria-modal="true"
        aria-labelledby={labelledBy}   // 5) 제목 연결
        onClick={(e) => e.stopPropagation()}
      >
        {children}
      </div>
    </div>,
    document.body
  );
}
```

그리고 `Modal.Header`에 `id`를 받아 붙이고, 쓰는 쪽에서 그 `id`를 `labelledBy`로 넘깁니다.

```tsx
<Modal open={open} onClose={close} labelledBy="order-title">
  <Modal.Header id="order-title">예매하기</Modal.Header>
  ...
</Modal>
```

### 왜 중요한가

- **포커스 트랩**: 모달이 떠 있는데 Tab으로 뒤 배경 버튼이 눌리면, 사용자는 보이지도 않는 버튼을 조작하게 됩니다.
- **복원**: 키보드 사용자는 모달을 닫으면 "내가 어디 있었지?"를 잃습니다. 연 버튼으로 돌려줘야 흐름이 이어집니다.
- **`aria-labelledby`**: 스크린리더가 다이얼로그를 열 때 제목을 읽어줘야 무슨 창인지 압니다.
