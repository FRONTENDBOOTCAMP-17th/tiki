# 예매 관리 서버 페이지네이션 · 필터 · CSV 내보내기 (`bang/feat/final-sprint`)

> 배경: 예매 관리 화면이 판매자의 모든 주문을 한 번에 불러와 클라이언트에서 필터·정렬하고 있었다. 주문이 쌓일수록 무거워지는 구조라, 필터와 페이지네이션을 서버로 내리고 CSV 내보내기를 붙였다.

## 1. 변경/추가 파일

| 경로 | 내용 |
|---|---|
| `src/app/seller/ticketManagement/page.tsx` | 서버에서 필터·페이지네이션 적용, 현재 페이지만 조회 |
| `src/app/seller/ticketManagement/_components/OrderTable.tsx` | 필터/페이지 이동을 searchParams 기반으로, 체크박스 선택 + 내보내기 버튼 |
| `src/app/api/seller/orders/export/route.ts` (신규) | CSV 내보내기 라우트 |

## 2. 서버 페이지네이션 · 필터

URL을 상태로 쓰는 방향으로 바꿨다. `?page=N`, `?eventId=`, `?status=`를 서버 컴포넌트가 읽어 쿼리에 반영한다.

- 페이지당 20건, supabase `.range()` + `count: "exact"`로 총 건수를 받아 페이지 수 계산
- 이벤트/상태 필터는 화면의 `Select`에서 고르면 `router.push`로 searchParams만 갈아끼운다 (클라이언트 상태 아님). 페이지 이동도 `Link` 기반
- 소유권: 항상 판매자 소유 이벤트로 스코핑한다. `?eventId=`가 넘어와도 **소유 목록 안에 있을 때만** 그 이벤트로 좁히고, 아니면 전체 소유 이벤트로 되돌린다

목록 조회는 현재 페이지 20건만, 통계 카드(예매 수량·결제액)는 **필터 전체 기준**이어야 해서 별도 집계 쿼리를 하나 더 돌린다. 이 집계 쿼리는 합계에 필요한 `quantity, total_price, status` 3개 컬럼만 가져온다.

## 3. CSV 내보내기

`GET /api/seller/orders/export`

- `requireUserApi` 가드 + 판매자 소유 이벤트로 스코핑 (다른 판매자 주문은 못 뽑는다)
- 화면의 현재 필터(`eventId`, `status`)를 쿼리스트링으로 그대로 받아 **페이지네이션 없이 전체**를 내보낸다
- CSV는 외부 패키지 없이 직접 생성. 값에 콤마/따옴표/줄바꿈이 있으면 따옴표로 감싸고 내부 따옴표는 두 번으로 이스케이프
- 한글 엑셀에서 깨지지 않게 UTF-8 BOM(`﻿`)을 앞에 붙이고, `Content-Disposition: attachment; filename="orders.csv"`

### 3-1. 선택한 행만 내보내기

표에 체크박스를 달아, 고른 주문만 뽑을 수 있게 했다.

- 아무것도 선택하지 않으면 현재 필터 전체를 내보낸다 (기본 동작)
- 몇 개 체크하면 그 `order_id`들만 `?ids=`로 넘겨 그것만 내보낸다. 이때도 event 스코핑은 그대로라 남의 주문을 id로 찍어 뽑을 수 없다
- 선택은 현재 페이지 안에서만 이뤄진다. 페이지를 옮기거나 필터가 바뀌면 목록이 달라지므로 선택을 초기화한다 — `useEffect` 대신 렌더 중 이전 키와 비교해 리셋하는 방식으로, 불필요한 리렌더를 피했다

## 4. 한계 / 후속

- 키워드 검색은 서버가 아니라 **현재 페이지 안**에서만 걸린다. searchParams 필터는 이벤트/상태까지만 옮겼고, 검색까지 서버로 내리려면 디바운스와 폼 제출 흐름이 추가로 필요하다.
- 정렬(주문일/결제액 등)도 현재 페이지 기준이다. 전체 정렬은 서버 `order`로 내려야 한다.
- 체크박스 선택은 페이지를 넘어 유지되지 않는다. 여러 페이지에 걸친 선택 내보내기는 범위에서 뺐다.
