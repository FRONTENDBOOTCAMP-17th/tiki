# 예매 관리 서버 페이지네이션 · 필터 · CSV 내보내기 (`bang/feat/final-sprint`)

> 배경: 예매 관리 화면이 판매자의 모든 주문을 한 번에 불러와 클라이언트에서 필터·정렬하고 있었습니다. 주문이 쌓일수록 무거워지는 구조라, 필터와 페이지네이션을 서버로 옮기고 CSV 내보내기를 추가했습니다.

## 1. 변경/추가 파일

| 경로 | 내용 |
|---|---|
| `src/app/seller/ticketManagement/page.tsx` | 서버에서 필터·페이지네이션 적용, 현재 페이지만 조회 |
| `src/app/seller/ticketManagement/_components/OrderTable.tsx` | 필터/페이지 이동을 searchParams 기반으로, 체크박스 선택 + 내보내기 버튼 |
| `src/app/api/seller/orders/export/route.ts` (신규) | CSV 내보내기 라우트 |

## 2. 서버 페이지네이션 · 필터

URL을 상태로 사용하는 방향으로 정리했습니다. `?page=N`, `?eventId=`, `?status=`를 서버 컴포넌트가 읽어 쿼리에 반영합니다.

- 페이지당 20건, supabase `.range()` + `count: "exact"`로 총 건수를 받아 페이지 수를 계산합니다.
- 이벤트/상태 필터는 화면의 `Select`에서 고르면 searchParams만 갱신되고, 페이지 이동도 `Link` 기반입니다(클라이언트 상태로 들고 있지 않습니다).
- 소유권: 항상 판매자 소유 이벤트로 스코핑합니다. `?eventId=`가 넘어와도 소유 목록에 있을 때만 좁히고, 그렇지 않으면 전체 소유 이벤트로 되돌립니다.

목록은 현재 페이지 20건만 조회하고, 통계 카드(예매 수량·결제액)는 필터 전체 기준이어야 하므로 별도 집계 쿼리를 한 번 더 돌립니다. 이 집계 쿼리는 합계에 필요한 `quantity, total_price, status` 세 컬럼만 가져오도록 해 불필요한 데이터 전송을 줄였습니다.

## 3. CSV 내보내기

`GET /api/seller/orders/export`

- `requireUserApi` 가드 + 판매자 소유 이벤트로 스코핑해, 다른 판매자의 주문은 내보낼 수 없습니다.
- 화면의 현재 필터(`eventId`, `status`)를 쿼리스트링으로 받아 페이지네이션 없이 전체를 내보냅니다.
- CSV는 외부 패키지 없이 직접 생성합니다. 값에 콤마/따옴표/줄바꿈이 있으면 따옴표로 감싸고, 내부 따옴표는 두 번으로 이스케이프합니다.
- 한글 엑셀에서 깨지지 않도록 UTF-8 BOM을 앞에 붙이고, `Content-Disposition: attachment; filename="orders.csv"`로 내려줍니다.

### 3-1. 선택한 행만 내보내기

표에 체크박스를 두어, 원하는 주문만 골라 내보낼 수 있게 했습니다.

- 아무것도 선택하지 않으면 현재 필터 전체를 내보냅니다(기본 동작).
- 몇 개를 체크하면 그 `order_id`들만 `?ids=`로 전달해 해당 주문만 내보냅니다. 이때도 이벤트 스코핑은 그대로라, id를 임의로 넣어 남의 주문을 가져올 수 없습니다.
- 선택은 현재 페이지 안에서 이뤄지며, 페이지를 옮기거나 필터가 바뀌면 목록이 달라지므로 선택을 초기화합니다. 이 초기화는 렌더 중 이전 키와 비교하는 방식으로 처리해 불필요한 리렌더를 피했습니다.

## 4. 범위 메모

- 키워드 검색과 정렬(주문일/결제액 등)은 현재 페이지 안에서 동작합니다. 이벤트/상태 필터를 우선 서버로 옮겼고, 검색까지 서버로 내리는 작업은 이후 확장 사항으로 남겨두었습니다.
- 체크박스 선택은 페이지 단위입니다. 여러 페이지에 걸친 선택 내보내기는 이번 범위에서 제외했습니다.
