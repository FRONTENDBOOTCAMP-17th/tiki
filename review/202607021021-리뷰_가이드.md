# TiKi 17차 리뷰 — 해결 가이드 (2026-07-02)

리뷰에서 짚은 것들을 어떻게 고치면 되는지 순서대로 적었습니다. 우선순위는 ① 비-UUID 상세 500 → ② 정책 페이지 정리 → ③ sitemap 슬래시 → ④ 상세 generateMetadata → ⑤ 상수 일원화 순입니다.

---

## 1. [필수] 비-UUID `/[eventId]` 500 막기

### 왜 나는가

상세 페이지는 여러 쿼리를 `Promise.all`로 한꺼번에 부릅니다.

```ts
// 상세 page.tsx (개념)
const [event, slots, grades, reviews] = await Promise.all([
  getEventDetail(eventId),  // ← 여기만 UUID 가드 있음 (null 반환)
  getSlots(eventId),        // ← 가드 없음
  getGrades(eventId),       // ← 가드 없음
  getReviews(eventId),      // ← 가드 없음
]);
```

`getEventDetail`은 UUID가 아니면 null을 돌려주지만, 나머지 셋은 그대로 `.eq("event_id", "not-a-uuid")`를 던집니다. Postgres는 uuid 컬럼에 이상한 문자열이 들어오면 `22P02` 에러를 내는데, 이게 `if (error) throw error`에서 던져지면 `Promise.all`이 통째로 깨지면서 500이 됩니다.

### 어떻게 고치나 — 두 가지 방법

**방법 A (추천): 페이지 진입에서 먼저 걸러내기.** 이미 있는 `getEventDetail`이 null이면 그 자리에서 `notFound()`를 부르고, 나머지 쿼리는 아예 실행하지 않습니다. `/payment/[orderId]`가 쓰는 패턴과 똑같습니다.

```ts
// src/app/[eventId]/page.tsx
import { notFound } from "next/navigation";

export default async function EventDetailPage({ params }) {
  const { eventId } = await params;

  const event = await getEventDetail(eventId); // UUID 아니면 null
  if (!event) notFound();                       // 여기서 404로 정상 종료

  // event가 유효할 때만 나머지 조회 (여기선 eventId가 정상 UUID임이 보장됨)
  const [slots, grades, reviews] = await Promise.all([
    getSlots(eventId),
    getGrades(eventId),
    getReviews(eventId),
  ]);
  // ...
}
```

이러면 잘못된 주소는 깔끔하게 404(안내 화면)로 가고, 불필요한 쿼리 세 번도 아끼게 됩니다.

**방법 B: 형제 쿼리에도 같은 가드.** 상세를 병렬로 유지하고 싶다면, `getSlots`/`getGrades`/`getReviews` 앞에도 `getEventDetail`과 똑같은 UUID 검사를 넣어 빈 값을 돌려주게 합니다.

```ts
// src/lib/event/queries.ts
export async function getSlots(eventId: string) {
  if (!UUID_RE.test(eventId)) return [];   // getEventDetail과 동일한 가드
  const { data, error } = await supabase.from("slots").select("*").eq("event_id", eventId);
  if (error) throw error;
  return data;
}
```

둘 중 **방법 A가 더 낫습니다.** 상세가 없는데 슬롯·좌석을 굳이 조회할 이유가 없고, 404 안내까지 자연스럽게 붙으니까요.

## 2. [제안] 정책/안내 페이지 한 갈래로 합치기

지금 `/terms`(루트)와 `/info/terms`가 둘 다 있습니다. 사용자는 푸터를 통해 `/info/*`만 보는데 sitemap에는 루트 `/terms`·`/privacy`만 올라가 있어서 서로 어긋난 상태입니다.

**정리 순서**
1. 팀에서 정본을 하나 정합니다. 안내 레이아웃(탭·뒤로가기)이 붙은 `/info/*`가 사용자 경험상 더 완성돼 있으니 그쪽을 남기는 걸 추천합니다.
2. 루트 `/terms`·`/privacy`·`/support` 페이지 파일을 지우거나, `/info/*`로 `redirect()`만 하도록 바꿉니다.

```ts
// src/app/terms/page.tsx (남길 거라면 리다이렉트만)
import { redirect } from "next/navigation";
export default function TermsRedirect() {
  redirect("/info/terms");
}
```

3. `sitemap.ts`의 URL 목록을 실제 사용하는 `/info/*` 경로로 맞춥니다.

이러면 색인에 올라가는 URL과 사용자가 실제 도달하는 URL이 맞아떨어지고, 약관 텍스트도 한 곳에서만 관리하면 됩니다.

## 3. [제안] sitemap·robots 슬래시 중복

`https://...vercel.app//support`처럼 `//`가 생기는 건 base와 경로가 둘 다 슬래시를 갖기 때문입니다.

```ts
// sitemap.ts — base에서 끝 슬래시 제거
const baseUrl = "https://tiki-final.vercel.app"; // 끝에 / 붙이지 않기
const urls = ["/", "/category", "/info/terms"].map((path) => ({
  url: `${baseUrl}${path}`,   // baseUrl + "/..." → 슬래시 하나만
  lastModified: new Date(),
}));
```

robots도 마찬가지로 `sitemap: \`${baseUrl}/sitemap.xml\`` 로 두면 `//`가 사라집니다.

## 4. [제안] 상세 페이지 generateMetadata

```ts
// src/app/[eventId]/page.tsx
export async function generateMetadata({ params }) {
  const { eventId } = await params;
  const event = await getEventDetail(eventId);
  if (!event) return { title: "이벤트를 찾을 수 없습니다 - TiKi" };
  return {
    title: `${event.title} - TiKi`,
    description: event.summary ?? undefined,
    openGraph: { title: event.title, images: event.posterUrl ? [event.posterUrl] : [] },
  };
}
```

`getEventDetail`을 여기서도 부르지만, Next가 같은 요청 안의 fetch/DB 호출을 캐시해 주기 때문에 본문 렌더와 중복 조회가 되진 않습니다(같은 함수라면 `React.cache`로 감싸 두면 더 확실합니다).
