# TiKi 2차 리뷰 — 해결 가이드 (2026-06-09)

## 1. 빌드 복구 — 파일명 대소문자 통일 (최우선)

원인: `example/button/page.tsx`가 `@/components/Button`을 import 하는데 파일은 `button.tsx`.

### 1-1. 컴포넌트 파일을 PascalCase로 통일 (권장)
macOS는 대소문자만 다른 rename을 git이 못 알아채므로 **2단계**로 합니다.
```bash
# button.tsx → Button.tsx (임시 이름 경유)
git mv src/components/button.tsx src/components/Button.tmp.tsx
git mv src/components/Button.tmp.tsx src/components/Button.tsx

git mv src/components/footer.tsx src/components/Footer.tmp.tsx
git mv src/components/Footer.tmp.tsx src/components/Footer.tsx

git mv src/components/navigation.tsx src/components/Navigation.tmp.tsx
git mv src/components/Navigation.tmp.tsx src/components/Navigation.tsx
```
그리고 import도 맞춥니다.
```ts
// example/button/page.tsx  → 이미 @/components/Button 이므로 OK
// example/navigation/page.tsx
import Navigation from "@/components/Navigation";   // n → N
// layout이나 페이지에서 footer를 쓴다면
import Footer from "@/components/Footer";
```

### 1-2. 확인
```bash
npx tsc --noEmit     # 0건이어야 함 (casing 에러 사라짐)
npm run build        # 성공해야 함
```
> 규칙으로 못 박기: README "폴더 구조/컨벤션"에 **"컴포넌트 파일은 PascalCase(`Button.tsx`), 스타일은 `Button.styles.ts`"** 라고 적어두면 다음 사고를 막습니다.

## 2. 라우트는 `page.tsx`로

빈 스캐폴딩 파일을 실제 페이지로 바꿀 때:
```
src/app/login/login.tsx      ❌ 라우트 아님
src/app/login/page.tsx       ✅ /login 으로 열림
```
보조 컴포넌트(페이지가 아닌 것)는 라우트로 오해되지 않게 `_`를 붙인 폴더에:
```
src/app/login/_components/LoginForm.tsx   ← 라우트 무시됨(좋음)
```
`productDetail.tsx/` 폴더는 이렇게 정리:
```bash
# 폴더 이름에 .tsx 가 붙은 잘못된 구조 제거 후
src/app/product/[id]/page.tsx   # 상세는 보통 동적 라우트로
```

> 팁: 지금 35개 빈 파일을 한 번에 다 바꿀 필요는 없습니다. **실제로 만들 페이지부터** `page.tsx`로 추가하세요. 안 쓰는 빈 파일은 지워도 됩니다(나중에 헷갈림 방지).

## 3. Button이 Spinner 재사용

```tsx
// Button.tsx
import Spinner from "./Spinner";

{loading && <Spinner size="sm" className="text-current" />}
```
이렇게 하면 스피너 모양을 Spinner 한 곳에서만 관리합니다.

## 5. (선택) 작은 다듬기

- `layout.tsx` metadata: `title: "TiKi"`, `description`도 서비스 소개로.
- Navigation active: 하위 경로까지 잡으려면
  ```ts
  const active = menu.href === "/" ? path === "/" : path.startsWith(menu.href);
  ```
- Button `outline` variant 기본 글자색을 `text-gray-700`으로(흰 배경 가독성).
- Navigation의 인라인 SVG 8개 → `components/icons.tsx`로 분리하거나 `lucide-react` 도입.
