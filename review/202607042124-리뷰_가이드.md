## 3. `getSlots`/`getGrades`/`getReviews` 내부에도 UUID 가드 넣기

## 4. 상세 페이지에 `generateMetadata` 추가

지금 `sitemap.ts`·`robots.ts`·`metadataBase`는 다 있는데, 실제로 검색 결과나 카카오톡 링크 공유에서 각 공연이 자기 이름으로 뜨려면 상세 페이지에 `generateMetadata`가 있어야 합니다. `getEventDetail`에 이미 `title`·`description`·`thumbnail`이 다 있으니 그대로 가져다 쓰면 됩니다.

```ts
// src/app/[eventId]/page.tsx
import type { Metadata } from "next";

export async function generateMetadata({
  params,
}: {
  params: Promise<{ eventId: string }>;
}): Promise<Metadata> {
  const { eventId } = await params;
  if (!EVENT_ID_RE.test(eventId)) return {};

  const event = await getEventDetail(eventId);
  if (!event) return {};

  return {
    title: event.title,
    description: event.description?.slice(0, 100) ?? undefined,
    openGraph: {
      title: event.title,
      description: event.description?.slice(0, 100) ?? undefined,
      images: event.thumbnail ? [event.thumbnail] : undefined,
    },
  };
}
```

`getEventDetail`을 상세 페이지 렌더용으로 또 한 번 부르는 셈이라 조회가 두 번 나가는데, 부담되면 `React.cache`로 감싸 같은 요청 안에서는 한 번만 조회되게 할 수 있습니다(17차 리뷰에서도 같은 얘기를 했던 부분입니다).

## 5. `robots.ts`에 `/example` disallow 추가

`/example/*`는 컴포넌트 문서용 데모 페이지인데 지금 그대로 프로덕션 라우트로 빌드되고 있고, `robots.ts`의 disallow 목록에는 빠져 있습니다.

```ts
// src/app/robots.ts
export default function robots(): MetadataRoute.Robots {
  const baseUrl = "https://tiki-final.vercel.app";

  return {
    rules: {
      userAgent: "*",
      allow: "/",
      disallow: ["/admin", "/mypage", "/seller", "/api", "/payment", "/example"],
    },
    sitemap: new URL("/sitemap.xml", baseUrl).toString(),
  };
}
```

시간이 더 있다면 `next.config.ts`에서 프로덕션 빌드 시 `/example` 라우트 자체를 제외하는 방법도 있지만, 마감이 사흘 남은 지금은 robots 한 줄 추가가 가장 가성비 좋은 선택입니다.
